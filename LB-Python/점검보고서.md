## 2. 잘 되어 있는 부분

- **라우팅 구조**: v1 기반 API, diet_analyzer 모듈 분리 적절함.
- **예외 처리**: 404 한글 메시지, 이미지 타입/빈 파일 검증 등 라우터 단 검증이 잘 되어 있음.
- **스키마**: Pydantic으로 요청/응답 규격이 명확함.
- **이미지 처리**: 리사이즈(512), base64 인코딩으로 비용·용량 관리 고려됨.
- **린트**: 현재 린터 에러 없음.

## 3. 개선이 필요한 부분

### 3.1 설정 및 환경 변수

- **문제**: `config.py`에서 `OPENAI_API_KEY`가 없어도 앱이 기동되고, 첫 요청 시 에러가 남.
- **권장**: 앱 기동 시 필수 환경 변수 검사 (예: `OPENAI_API_KEY`, `AI_MODEL` 기본값 여부).

### 3.2 미사용 코드 (Dead Code) - 해결

- **문제**: `app/api/v1/diet_analyzer/service_gemini.py`가 어디에서도 임포트되지 않음.  
  라우터는 `service.py`(OpenAI)만 사용 중.
- **권장**:  
  - Gemini를 쓰지 않을 예정이면 제거하거나 `docs/` 등에 “미사용 백엔드”로 문서화.  
  - Gemini를 쓸 예정이면 `requirements.txt`에 `google-genai` 추가 후, 라우터/설정에서 선택 가능하도록 연결.

### 3.3 의존성 버전

- **문제**: `requirements.txt`에 패키지 버전이 없음 → 재현 가능한 빌드가 어렵고, 향후 호환성 이슈 가능.
- **권장**: 최소한 `fastapi`, `uvicorn`, `openai` 등 핵심 패키지는 버전 고정 (예: `fastapi>=0.109,<0.110`).

### 3.4 보안 및 에러 노출

- **문제**: `router.py` 500 에러 시 `detail=f"식단 분석 중 오류가 발생했습니다: {str(e)}"` 로 내부 예외 메시지를 그대로 클라이언트에 반환함.
- **권장**: 운영 환경에서는 `str(e)` 노출하지 않고, 로그에만 상세 내용 기록. 클라이언트에는 일반 메시지만 반환.

### 3.5 테스트

- **문제**: `test_*.py` 또는 `*_test.py` 없음.
- **권장**:  
  - 라우터: 이미지 업로드 성공/실패(400, 500)에 대한 pytest + TestClient.  
  - 서비스: `process_and_encode_image`, `analyze_diet_image`(모킹) 단위 테스트 추가 검토.

### 3.6 타입 힌트

- **문제**: `ai_utils.generate(messages: list, response_format)` 에서 `list`만 사용, `response_format` 타입 없음.
- **권장**: `list[dict]` 또는 TypedDict, `response_format`은 `type[BaseModel]` 등으로 구체화하면 유지보수에 유리.

### 3.7 .gitignore - 해결

- **현재**: venv 관련 + `.env`, `data` 무시.
- **권장**: `__pycache__/`, `*.pyc`, `.env`, `*.egg-info/`, `dist/`, `data/` 등은 유지.  
  `Include`, `Lib`, `Scripts` 등은 보통 venv 폴더 이름이므로, 상위에 `venv/` 가 있으면 `venv/` 만 있어도 충분할 수 있음 (선택).
